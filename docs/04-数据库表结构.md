# Course_Downloader — 数据库表结构

> 版本：v1.0 | 日期：2026-05-14
> 引擎：SQLite 3 + SQLAlchemy 2.0 (async)
> 文件：`data/course_downloader.db`

---

## 总览：4 张表

```mermaid
erDiagram
    platforms ||--o{ accounts : "1:N"
    accounts ||--o{ tasks : "1:N"
```

| 表名 | 用途 | 核心字段 |
|------|------|---------|
| `platforms` | 平台配置（腾讯会议/小鹅通/B站/小红书/今日头条/抖音） | name, display_name, enabled, output_dir |
| `accounts` | 各平台下的账号 | platform_id, name, cookie_file, is_active |
| `tasks` | 下载任务（每个平台每条内容一条记录） | account_id, platform, resource_id, title, status, video_path... |
| `api_keys` | 第三方 API Key（SiliconFlow 等） | name, key_value, provider, is_active |

## 表 1：platforms

```sql
CREATE TABLE platforms (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    name          VARCHAR(50)  NOT NULL UNIQUE,  -- tencent_meeting, xiaoe, bilibili...
    display_name  VARCHAR(50)  NOT NULL,          -- 腾讯会议, 小鹅通, B站...
    enabled       BOOLEAN      DEFAULT 1,
    output_dir    VARCHAR(500),                    -- 该平台的输出目录
    sort_order    INTEGER      DEFAULT 0,         -- 排序
    created_at    DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME     DEFAULT CURRENT_TIMESTAMP
);
```

## 表 2：accounts

```sql
CREATE TABLE accounts (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    platform_id   INTEGER      NOT NULL REFERENCES platforms(id) ON DELETE CASCADE,
    name          VARCHAR(100) NOT NULL,           -- 账号别名（如「熊子熠」「Edward Fang」）
    cookie_file   VARCHAR(200),                     -- cookie 存储路径（相对 browser_profiles/）
    is_active     BOOLEAN      DEFAULT 0,           -- 当前活跃账号
    last_login    DATETIME,                          -- 最近一次登录成功时间
    created_at    DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME     DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(platform_id, name)
);

CREATE INDEX idx_accounts_platform ON accounts(platform_id);
```

**说明：**
- 一个平台可以有多个账号（如熊子熠的腾讯会议账号 + Edward Fang 的腾讯会议账号）
- `is_active` 同一平台只能有一个活跃账号
- Cookie 存储在 `data/browser_profiles/{platform}/{account_id}/`，由 Playwright 管理

## 表 3：tasks

```sql
-- 状态枚举（SQLAlchemy 中为 Python Enum，SQLite 中存文本）
-- pending / running / partial / done / failed / cancelled / fatal / updated / new

CREATE TABLE tasks (
    id                INTEGER PRIMARY KEY AUTOINCREMENT,
    account_id        INTEGER      REFERENCES accounts(id) ON DELETE SET NULL,
    platform          VARCHAR(50)  NOT NULL,       -- 冗余字段，方便按平台过滤
    resource_id       VARCHAR(200),                 -- 平台唯一标识（v_xxxxxxxx / record_id）
    title             VARCHAR(500),                  -- 课程/视频标题
    content_type      VARCHAR(20)  DEFAULT 'video', -- video / article
    url               VARCHAR(500),                 -- 详情页 URL

    -- 任务状态
    status            VARCHAR(20)  DEFAULT 'pending',

    -- 各子步骤独立状态
    video_status      VARCHAR(20)  DEFAULT 'pending',   -- pending/running/done/failed/skipped
    subtitle_status   VARCHAR(20)  DEFAULT 'pending',
    transcript_status VARCHAR(20)  DEFAULT 'pending',
    article_status    VARCHAR(20)  DEFAULT 'pending',

    -- 文件输出路径
    video_path        VARCHAR(500),
    subtitle_path     VARCHAR(500),
    transcript_path   VARCHAR(500),
    article_path      VARCHAR(500),

    -- 错误记录
    error_message     TEXT,                          -- 最后一次错误信息
    retry_count       INTEGER      DEFAULT 0,        -- 已重试次数

    -- 进度元数据（用于增量更新检测）
    duration_seconds  INTEGER,                        -- 视频时长（秒）
    file_size_bytes   INTEGER,                        -- 文件大小
    server_updated_at DATETIME,                       -- 服务端课程最后更新时间
    file_hash         VARCHAR(64),                     -- 文件 SHA256（可选）

    -- 时间
    created_at        DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at        DATETIME     DEFAULT CURRENT_TIMESTAMP,
    downloaded_at     DATETIME,                       -- 下载完成时间

    UNIQUE(platform, resource_id)
);

CREATE INDEX idx_tasks_status        ON tasks(status);
CREATE INDEX idx_tasks_platform      ON tasks(platform);
CREATE INDEX idx_tasks_platform_status ON tasks(platform, status);
CREATE INDEX idx_tasks_resource      ON tasks(resource_id);
```

### task 状态机

```
        ┌─────────┐
        │  new    │  ← 扫描发现的新课程
        └────┬────┘
             │ 用户确认下载
             ▼
        ┌─────────┐
        │ pending │  ← 等待队列
        └────┬────┘
             │ 调度
             ▼
        ┌─────────┐
        │ running │  ← 正在下载
        └────┬────┘
             │
    ┌────────┼────────────┐
    ▼        ▼            ▼
┌──────┐ ┌──────┐   ┌─────────┐
│ done │ │ fail │   │ partial │  ← 部分成功（视频下了但ASR失败）
└──────┘ └──────┘   └─────────┘
    │        │
    │        │ 超过最大重试次数
    │        ▼
    │   ┌─────────┐
    │   │  fatal  │  ← 不再自动重试
    │   └─────────┘
    │
    │ 下次扫描时检测到课程更新
    ▼
┌─────────┐
│ updated │  ← 标记「已更新」
└─────────┘
    │ 用户可以手动触发重新下载
    ▼
  pending
```

### 子步骤状态独立

```python
video_status = done         # 视频下载成功
subtitle_status = done      # 字幕下载成功
transcript_status = failed  # ASR 转写失败（不影响其他状态）

# 任务整体状态
task_status = partial       # 因为有子步骤失败
```

## 表 4：api_keys

```sql
CREATE TABLE api_keys (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        VARCHAR(100) NOT NULL,         -- 用户自定义名称
    key_value   VARCHAR(500) NOT NULL,         -- API Key（明文存储）
    provider    VARCHAR(50),                    -- siliconflow / aliyun / volcengine
    is_active   BOOLEAN      DEFAULT 0,         -- 当前启用的 Key
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_api_keys_provider ON api_keys(provider);
```

**安全说明：**
- Key 明文存 SQLite（个人工具，不做加密）
- 日志中自动脱敏：`mask_key("sk-qvjfkfcb...hausrh")` → `sk-q...hsrh`
- API 返回时只显示后 4 位
- 前端输入框用 `type="password"`

## SQLAlchemy ORM 模型（Python）

```python
import enum
from datetime import datetime
from sqlalchemy import (
    Column, Integer, String, Boolean, DateTime,
    ForeignKey, Text, Enum, create_engine, Index
)
from sqlalchemy.orm import DeclarativeBase, relationship

class Base(DeclarativeBase):
    pass

class TaskStatus(str, enum.Enum):
    PENDING   = "pending"
    RUNNING   = "running"
    PARTIAL   = "partial"
    DONE      = "done"
    FAILED    = "failed"
    CANCELLED = "cancelled"
    FATAL     = "fatal"
    UPDATED   = "updated"
    NEW       = "new"

class StepStatus(str, enum.Enum):
    PENDING = "pending"
    RUNNING = "running"
    DONE    = "done"
    FAILED  = "failed"
    SKIPPED = "skipped"

class Platform(Base):
    __tablename__ = "platforms"
    id = Column(Integer, primary_key=True)
    name = Column(String(50), unique=True, nullable=False)
    display_name = Column(String(50), nullable=False)
    enabled = Column(Boolean, default=True)
    output_dir = Column(String(500))
    sort_order = Column(Integer, default=0)
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    accounts = relationship("Account", back_populates="platform", cascade="all, delete-orphan")

class Account(Base):
    __tablename__ = "accounts"
    id = Column(Integer, primary_key=True)
    platform_id = Column(Integer, ForeignKey("platforms.id", ondelete="CASCADE"), nullable=False)
    name = Column(String(100), nullable=False)
    cookie_file = Column(String(200))
    is_active = Column(Boolean, default=False)
    last_login = Column(DateTime)
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    platform = relationship("Platform", back_populates="accounts")
    tasks = relationship("Task", back_populates="account")
    __table_args__ = (UniqueConstraint("platform_id", "name"),)

class Task(Base):
    __tablename__ = "tasks"
    id = Column(Integer, primary_key=True)
    account_id = Column(Integer, ForeignKey("accounts.id", ondelete="SET NULL"))
    platform = Column(String(50), nullable=False)
    resource_id = Column(String(200))
    title = Column(String(500))
    content_type = Column(String(20), default="video")
    url = Column(String(500))
    status = Column(Enum(TaskStatus), default=TaskStatus.PENDING)
    # 子步骤状态
    video_status = Column(Enum(StepStatus), default=StepStatus.PENDING)
    subtitle_status = Column(Enum(StepStatus), default=StepStatus.PENDING)
    transcript_status = Column(Enum(StepStatus), default=StepStatus.PENDING)
    article_status = Column(Enum(StepStatus), default=StepStatus.PENDING)
    # 文件路径
    video_path = Column(String(500))
    subtitle_path = Column(String(500))
    transcript_path = Column(String(500))
    article_path = Column(String(500))
    # 错误和重试
    error_message = Column(Text)
    retry_count = Column(Integer, default=0)
    # 进度元数据
    duration_seconds = Column(Integer)
    file_size_bytes = Column(Integer)
    server_updated_at = Column(DateTime)
    file_hash = Column(String(64))
    # 时间
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    downloaded_at = Column(DateTime)
    account = relationship("Account", back_populates="tasks")
    __table_args__ = (
        UniqueConstraint("platform", "resource_id"),
        Index("idx_tasks_status", "status"),
        Index("idx_tasks_platform", "platform"),
        Index("idx_tasks_platform_status", "platform", "status"),
    )

class APIKey(Base):
    __tablename__ = "api_keys"
    id = Column(Integer, primary_key=True)
    name = Column(String(100), nullable=False)
    key_value = Column(String(500), nullable=False)
    provider = Column(String(50))
    is_active = Column(Boolean, default=False)
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
```

## 设计说明

### 为什么用 Enum 不用 TEXT

```python
# ❌ 错误：字符串字段，可以写入任何值
status = Column(String)

# ✅ 正确：枚举约束，只能写入定义的值
class TaskStatus(enum.Enum):
    PENDING = "pending"
    ...

status = Column(Enum(TaskStatus))
```

SQLite 本身不支持 ENUM 类型，但 SQLAlchemy 的 `Enum` 类型会在应用层做校验。写入无效值时抛出异常，而不是静默写入脏数据。

### 为什么 tasks 表冗余 platform 字段

`task.platform` 和 `task.account.platform.name` 是冗余的。冗余理由：
- 90% 的查询需要按平台过滤
- 避免每次查询都 JOIN accounts 表
- 数据一致性由应用层保证

### 为什么不做数据库加密

个人工具 + 本地运行 + 单用户。加密增加了「Key放在哪」的难题（鸡生蛋问题）。日志脱敏 + .gitignore 已经覆盖主要风险。
