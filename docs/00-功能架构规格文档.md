## 第一章：项目概述

### 1.1 项目定位

Course_Downloader 是一个**多平台内容离线下载与管理工具**，统一管理各平台课程视频、图文内容的下载、转码、文字转写和归档。

### 1.2 支持平台

| 平台 | 类型 | 内容 | 优先级 | 技术风险 |
|------|------|------|--------|---------|
| 腾讯会议 | 在线会议 | 视频 + 逐字稿 + 纪要 + 时间轴 | P0 核心 | 🟢 低 |
| 小鹅通 | 知识付费 | 视频（HLS TS + AES 解密） | P0 核心 | 🟢 低 |
| B站 | 视频平台 | 视频 + CC 字幕 | P1 重要 | 🟡 中 |
| 今日头条 | 资讯平台 | 收藏文章 + 视频 | P2 一般 | 🟡 中 |
| 抖音 | 短视频 | 收藏视频 | P3 补充 | 🔴 高 |
| 小红书 | 社交平台 | 图文笔记 + 教程视频 | P3 补充 | 🔴 高 |

### 1.3 核心能力

```
✅ 视频下载 — 各平台自动适配下载方式
✅ 视频转码 — TS → MP4（H.265，ffmpeg）
✅ 文字转写 — ASR 自动转写（SiliconFlow API，带时间戳）
✅ 增量检测 — 新课自动识别、更新课程提示重下
✅ 多账号管理 — 各平台独立账号配置
✅ 断点续传 — 逐项独立状态
✅ Web 管理界面 — 仪表盘 / 任务中心 / 设置 / 日志
```

```
├─────────────────────────────────────────────┤

└─────────────────────────────────────────────┘
```

### 2.2 插件架构

```python
# 所有平台必须实现
class BasePlatform(ABC):
    @abstractmethod
    def login(self) -> LoginResult: ...
    @abstractmethod
    def fetch_list(self, page_token=None) -> FetchResult: ...
    @abstractmethod
    def get_metadata(self, item) -> dict: ...

# 可选能力（Mixin）
class VideoCapable(ABC):
    @abstractmethod
    def download_video(self, item, output, quality="720p") -> DownloadResult: ...

class ArticleCapable(ABC):
    @abstractmethod
    def download_article(self, item, output) -> DownloadResult: ...

class SubtitleCapable(ABC):
    @abstractmethod
    def download_subtitle(self, item, language="zh") -> str: ...
```

## 第三章：功能规格

### 3.1 多账号管理

- 每个平台可配置多个账号，各自独立登录态
- 账号切换时自动切换 Chrome Profile
- 登录 Cookie 持久化到 Profile 目录（非 pickle 文件）
- 登录检测做 API 往返验证，不靠 URL 字符串匹配

### 3.2 任务队列策略

```
队列模型：
  平台内串行     — 同一平台的任务逐个处理
  平台间并发     — 最多 2 个平台同时运行
  实例管理       — 每个平台独立 Playwright 实例
                 最大同时 2 个实例（实测 687MB/实例）
                 闲置超过 5 分钟自动关闭
```

### 3.3 WebSocket 推送格式

```json
// 服务端 → 客户端
{"type":"task_update", "task_id":1, "step":"downloading",
 "step_index":1, "step_total":3, "step_progress":65,
 "total_progress":22, "message":"下载中 (65%)"}

{"type":"task_done", "task_id":1}
{"type":"task_error", "task_id":1, "step":"asr",
 "error":"API 超时"}
```

### 3.4 增量检测

```python
def should_redownload(local_task, server_course) -> bool:
    """判断是否需要重新下载"""
    # 服务器端更新时间 > 本地下载时间 → 内容可能更新
    if server_course.updated_at > local_task.downloaded_at:
        return True
    # 视频时长变化 → 大概率重新录制
    if server_course.duration_seconds != local_task.duration_seconds:
        return True
    return False
```

### 3.5 断点续传粒度

| 子步骤 | 状态字段 | 断点续传 |
|--------|---------|---------|
| 视频下载 | `video_status` | ✅ HTTP Range |
| 字幕下载 | `subtitle_status` | ✅ 重启可跳过 |
| ASR 转写 | `transcript_status` | ✅ 每段保存 .part |
| 文章抓取 | `article_status` | ✅ 重启可跳过 |

### 4.1 任务状态机

```
NEW ──→ PENDING ──→ RUNNING ──┬──→ DONE
                               ├──→ PARTIAL ──→ 手动重试失败步骤 ──→ PENDING
                               └──→ FAILED ──→ 重试≥3次 ──→ FATAL
```

## 第五章：错误处理

### 5.1 三层兜底

```
L1（操作级）  : 3 次重试（0s→5s→15s）
L2（任务级）  : 子步骤独立失败 + 任务结束整体重试
L3（系统级）  : 标记 FATAL + 通知用户 + 提供重试/跳过按钮
```

### 5.2 实时监测

```
WebSocket     : 每 1 秒推送
卡死检测       : 5 分钟无进度 → kill + 重启
浏览器无响应  : 10 分钟 → 关闭 + 重启
下载无数据     : 30 秒 → 断线重连 + 续传
ASR 无响应     : 60 秒 → 取消 + 重新提交
```
