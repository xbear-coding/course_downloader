🎯 设计说明
1. 子步骤可视化 — 每个任务拆分为独立步骤显示进度条，用户清楚知道当前卡在哪一步。
2. 「部分完成」状态 — 视频下完了但 ASR 失败的中间态，提供「重试转写」按钮而非整任务重跑。
3. 批量操作 — 左侧复选框支持多选 + 批量删除/重试/取消。
4. 过滤搜索 — 按状态/平台/关键词快速筛选大量任务。
5. 操作按钮 —「重试」= 针对失败子步骤，「打开」= 打开下载目录查看文件。

🎯 设计说明
1. Tab 切换分类 — 设置项用 Tab 分 4 类，避免一个页面太长。
2. API Key 仅显示后 4 位 — 安全性考虑。扩展卡可展开看完整 Key。
3. 测试按钮 — 配置完 Key 后一键验证有效性，避免用到无效 Key 时才发现。
4. 磁盘空间显示 — 实时显示剩余空间，警告 < 1GB 时标记红色。
5. 并发浏览器数限制 — 防止用户同时开太多实例导致内存不足。

🎯 交互说明
用户触发「扫描更新」后的反馈闭环：
点击 → 弹窗显示「正在扫描...」加载态 → 完成后显示本次扫描结果（新增/更新/跳过）
用户确认 → 后台自动加入下载队列 → WebSocket 推送进度
整个过程无需手动勾选每条课程。

🎯 交互链路说明
